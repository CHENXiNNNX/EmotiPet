一、需要判定的数据格式(格式对应:触摸-压力-陀螺仪-光敏-声音，xxxx的二进制形式)
1. 00000 --- 五个设备无数据变化
2. 00001 --- 声音变化
3. 以此类推
注意:每一个数据需要新旧状态的存储效果，因此每一个数据均采用队列存储，拟定存储 20 个，取第一位作为新状态，其它位作为旧状态
建议：只储存一个数据，当三分钟（时间待定）内没有检测到数据则关闭，如有变化则重新计时

二、数据判断条件
1. 触摸
当检测到触摸传感器的数据发生变化，达到设定的阈值的时候，即被触摸的时候
2. 压力
当检测到压力传感器的数据发生变化，达到设定的阈值的时候，即存在压力的时候
3. 陀螺仪
当检测到陀螺仪的数据发生变化，达到设定的阈值的时候，即摇晃，倒置等等
4. 光敏
当检测到光敏的数据发生变化，达到设定的阈值的时候，即明亮，黑暗等等
5. 声音
识别到环境存在人声，或者分贝大小超过阈值

三、需要的上传数据(服务器可控，1:打开，0:关闭)
1. 触摸
2. 压力
3. 陀螺仪
4. 光敏
5. 摄像头
注意:数据上传的格式位:触摸-压力-陀螺仪-光敏-摄像头，xxxx的二进制形式(服务器需使用)

(1) 数据上传
  {
    "type": "transport_info",
    "command": "00001",                      // 00001  对应摄像头打开进行上传，以此类推
    "data":{
      "touch": 0,                          // 触摸状态，0=未触摸，1=触摸
      "pressure": 0.0,                     // 压力值，浮点数，单位：Pa
      "gyroscope": {                       // 陀螺仪数据
        "x": 0.0,                          // X轴角速度，单位：deg/s
        "y": 0.0,                          // Y轴角速度，单位：deg/s
        "z": 0.0                           // Z轴角速度，单位：deg/s
      },
      "photosensitive": 0                  // 光敏值，整数，单位：lux                          
    }
  }
注意:command字段为5位字符串，每一位对应一个传感器：触摸-压力-陀螺仪-光敏-摄像头，0表示关闭，1表示开启。例如"00001"表示仅摄像头开启

(3) 数据接收
  {
    "type": "recv_info",
    "command": "00001",      // 00001  对应摄像头打开进行上传，以此类推
  }
注意:command字段为5位字符串，每一位对应一个传感器：触摸-压力-陀螺仪-光敏-摄像头，0表示关闭，1表示开启。例如"00001"表示仅摄像头开启

(4) 运动数据接收
  {
    "type": "mov_info",
    "data":{
        "servo_01": {
        "start_time": "2025-03-12T19:00:00Z", // 运动的起始时间，ISO 8601格式
        "angle": 90,                          // 舵机变换到指定角度，整数，范围：0-180（暂定限位，后续需要可扩展为每个舵机限制）
        "duration": 1000                      // 运动持续时间，整数，单位：毫秒
      },
      "servo_02": {
        "start_time": "2025-03-12T19:00:00Z",
        "angle": 90,
        "duration": 1000                      
      }         
      ...
    }     
  }
